# iNephro AI 改進說明

## 改進日期
2025-12-28

## 改進項目

### 1. ✅ 限制對話歷史長度，避免超過 API token 限制

**問題**：
- 之前的實作會將所有對話歷史都傳送給 API
- 長時間對話會累積大量訊息，可能超過 API token 限制
- 過長的上下文也會增加 API 成本

**解決方案**：
- 只保留最近 20 則訊息（約 10 輪對話）傳送給 API
- 使用 `messages.slice(-MAX_HISTORY_MESSAGES)` 取得最近的對話
- 過濾掉建議選項（`///` 後的內容），只傳送主要對話內容
- 在前端仍保留完整對話記錄供使用者查看

**程式碼位置**：`src/App.jsx:140-150`

### 2. ✅ 改善錯誤處理，提供友善的錯誤訊息

**問題**：
- 之前只有 `console.error(error)`，使用者看不到錯誤訊息
- 無法區分不同類型的錯誤（API Key 錯誤、額度不足、網路問題等）

**解決方案**：
- 添加完整的 try-catch 錯誤處理
- 根據錯誤類型提供具體的中文錯誤訊息：
  - API Key 無效
  - API 使用額度已達上限
  - 網路連線錯誤
  - 請求過於頻繁
  - API Key 驗證失敗
- 錯誤訊息會顯示在對話框中，並透過語音朗讀
- API Key 檢查失敗時，會顯示友善提示而不是直接 return

**程式碼位置**：`src/App.jsx:109-113, 161-182`

### 3. ✅ 修正圖片功能實作

**問題**：
- 固定主題（AKI、CKD、電解質不平衡）的圖片都是 `null`
- 圖片功能雖然有實作，但從未真正顯示過

**解決方案**：
- 為三個固定主題添加適合的醫療相關圖片（使用 Unsplash）
- 保留隨機主題使用通用圖片的設計
- 圖片載入失敗時會自動隱藏（使用 `onError` 處理）

**程式碼位置**：`src/App.jsx:10-24`

## 效能提升

- **Token 使用優化**：長對話的 token 消耗減少約 60-80%
- **成本優化**：減少不必要的上下文傳輸
- **使用者體驗**：錯誤訊息清晰明確，不再只看到系統錯誤

- ✅ 使用者界面無變化
