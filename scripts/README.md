# iNephro 快取預熱腳本

## 📌 功能說明

這個腳本會自動預先載入常見問題到 Upstash Redis 快取，提升用戶體驗並降低 API 成本。

## 🚀 使用方式

### 方式 1：使用 npm script（推薦）

```bash
npm run warmup
```

### 方式 2：直接執行

```bash
node scripts/warmup-cache.js
```

## ⚙️ 環境設定

腳本會使用以下 API 端點（按優先順序）：

1. 環境變數 `VITE_API_BASE_URL`
2. 預設值：`https://i-nephro.vercel.app`（正式環境）

### 本地測試

如果想在本地開發環境測試：

```bash
# 先啟動本地開發伺服器
npm run dev

# 在另一個終端視窗執行
VITE_API_BASE_URL=http://localhost:5173 npm run warmup
```

### 正式環境

直接執行即可使用預設的正式環境 URL：

```bash
npm run warmup
```

## 📋 問題清單

常見問題定義在 `common-questions.json`，包含 37 個常見腎臟科問題：

- 慢性腎臟病 (CKD) 相關
- 急性腎損傷 (AKI)
- 透析治療（血液透析、腹膜透析）
- 常見症狀（蛋白尿、血尿、水腫）
- 檢驗數值（肌酸酐、eGFR、BUN）
- 飲食注意事項
- 藥物影響

## ⏰ 建議執行時機

### 1. 知識庫更新後

當 OpenAI Assistant 的知識庫更新時，執行一次預熱：

```bash
npm run warmup
```

### 2. 定期維護

建議每月執行一次，確保快取保持最新：

```bash
# 可以設定 cron job 自動執行
0 0 1 * * cd /path/to/inephro && npm run warmup
```

### 3. 首次部署

新環境部署後，立即執行預熱：

```bash
npm run warmup
```

## 🔧 進階設定

### 調整請求間隔

預設每個請求間隔 2 秒，如需調整請編輯 `warmup-cache.js`：

```javascript
// 每個請求之間延遲 2 秒
await delay(2000);  // 改為其他數值（毫秒）
```


## ❓ 常見問題

**Q: 執行需要多久？**
A: 37 個問題，每個間隔 2 秒 + API 回應時間（約 3-5 秒），總共約 4-5 分鐘。

**Q: 失敗了怎麼辦？**
A: 查看 `warmup-log.json` 中的失敗原因，通常是網路問題或 API 限制，重新執行即可。

**Q: 需要每次部署都執行嗎？**
A: 不需要。快取有 30 天有效期，只有在知識庫更新或快取過期時才需要重新預熱。

**Q: 會產生額外費用嗎？**
A: 會調用 37 次 OpenAI API，但這些快取會持續 30 天，長期來看大幅降低成本。
